# MANUAL DEL DESARROLLADOR - Juego Memes 8M

## 👨‍💻 Introducción

Este manual está dirigido a desarrolladores que deseen contribuir, mantener o extender el proyecto **Juego Memes 8M**. El proyecto es una aplicación Java de consola que utiliza archivos de datos en formatos TXT, JSON y XML para gestionar un juego educativo sobre memes del 8M.

**Autores:** Javier, Victor, Alejandro  
**Fecha:** 08/03/2026  
**Asignatura:** Programación (DAW1)

---

## 🛠️ Configuración del Entorno de Desarrollo

### Requisitos

- **Java JDK 25** o superior
- **IDE recomendado:** Visual Studio Code, IntelliJ IDEA o Eclipse
- **Librerías:** 
  - `json-20251224.jar` para parseo JSON
  - JUnit 5 para testing (`junit-jupiter-api-5.9.2.jar`, `junit-jupiter-engine-5.9.2.jar`, `junit-platform-console-standalone-1.9.2.jar`, `hamcrest-core-1.3.jar`)

### Instalación

1. **Clona el repositorio:**
   ```bash
   git clone https://github.com/VictorRepollet/ProyectoMemes8M.git
   cd ProyectoMemes8M
   ```

2. **Verifica dependencias:**
   - Los JARs deben estar en `lib/`
   - Archivos de datos en `datos/`

3. **Configura el IDE:**
   - Añade `lib/json-20251224.jar` al classpath
   - Para tests, incluye todos los JARs de JUnit

## 🗂️ Arquitectura del Proyecto

### Estructura de Directorios

```
ProyectoMemes8M/
├── datos/                 # Archivos de datos de entrada
│   ├── memes.txt          # Lista de memes (formato TXT)
│   ├── realidades.json    # Realidades y opciones falsas (JSON)
│   └── soluciones.xml     # Soluciones correctas (XML)
├── fuentes/               # Código fuente principal
│   ├── JuegoMemes.java    # Clase principal (main)
│   ├── ComprobadorFicheros.java  # Verificación de archivos
│   ├── LeerFicheros.java  # Lectura/escritura de archivos
│   ├── Memes.java         # Modelo de datos básico
│   ├── MemesRealidades.java # Modelo extendido
│   ├── Puntuacion.java    # Modelo de puntuación
│   └── Puntuaciones.java  # Gestión de rankings
├── test/                  # Pruebas unitarias
│   ├── JuegoMemesTest.java
│   ├── ComprobadorFicherosTest.java
│   ├── LeerFicherosTest.java
│   └── PuntuacionesTest.java
├── lib/                   # Librerías JAR
├── compilados/            # Archivos .class (generado)
├── resultados/            # Archivos de salida
│   └── resultados.txt     # Ranking persistente
├── documentacion/         # Javadoc generado
└── [Archivos de documentación]
```

### Diagrama de Clases

- **JuegoMemes:** Punto de entrada, controla el flujo del juego
- **ComprobadorFicheros:** Utilidad para verificar existencia de archivos
- **LeerFicheros:** Maneja I/O de archivos (TXT, JSON, XML)
- **Memes:** Clase POJO para representar un meme básico
- **MemesRealidades:** Extiende Memes con realidad y opciones
- **Puntuacion:** Representa una entrada de puntuación
- **Puntuaciones:** Gestiona el ranking y persistencia

## 🚀 Compilación y Ejecución

### Compilación Básica

```bash
# Compilar fuentes principales
javac -cp "lib/json-20251224.jar" -d compilados fuentes/*.java

# Compilar con tests
javac -cp "lib/junit-jupiter-api-5.9.2.jar;lib/junit-jupiter-engine-5.9.2.jar;lib/hamcrest-core-1.3.jar;lib/json-20251224.jar;compilados" -d compilados test/*.java
```

### Ejecución

```bash
# Ejecutar aplicación principal
java -cp "compilados;lib/json-20251224.jar" JuegoMemes

# Ejecutar tests
java -jar lib/junit-platform-console-standalone-1.9.2.jar --scan-classpath -cp "compilados;lib/junit-jupiter-api-5.9.2.jar;lib/junit-jupiter-engine-5.9.2.jar;lib/hamcrest-core-1.3.jar;lib/json-20251224.jar"
```

### Scripts Combinados

```bash
# Compilar y ejecutar en un comando
javac -cp "lib/json-20251224.jar" -d compilados fuentes/*.java && java -cp "compilados;lib/json-20251224.jar" JuegoMemes
```

**Nota:** En Linux/Mac, reemplaza `;` por `:` en classpath.

## 🔧 Desarrollo y Contribución

### Estándares de Código

- **Lenguaje:** Java 11+ (compatible con JDK 25)
- **Estilo:** CamelCase para métodos/variables, PascalCase para clases
- **Documentación:** Javadoc en clases públicas y métodos
- **Testing:** Tests unitarios para toda funcionalidad nueva

### Flujo de Desarrollo

1. **Clona el repositorio**
2. **Crea una rama** para tu feature: `git checkout -b feature/nueva-funcionalidad`
3. **Implementa cambios** con tests
4. **Compila y ejecuta tests** localmente
5. **Genera Javadoc** si es necesario
6. **Commit y push** tus cambios
7. **Crea Pull Request**

### Agregar Nueva Funcionalidad

1. **Identifica la HU** en `Backlog.txt`
2. **Implementa en la clase correspondiente**
3. **Añade tests** en `test/`
4. **Actualiza documentación**

### Generar Documentación

```bash
# Generar Javadoc
javadoc -d documentacion -sourcepath fuentes -subpackages "" -cp "lib/json-20251224.jar"
```

## 🐛 Debugging y Troubleshooting

### Problemas Comunes

- **Error de classpath:** Verifica rutas a JARs
- **Archivos no encontrados:** Comprueba estructura de directorios
- **Tests fallan:** Asegura compilación previa de fuentes
- **Encoding issues:** Usa UTF-8 para archivos de datos

### Logs y Debugging

- El programa imprime mensajes en consola
- Para debugging avanzado, usa IDE con breakpoints
- Tests proporcionan feedback detallado
